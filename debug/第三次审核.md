#### ✅ P0-2：庄闲倍数逻辑完全缺失

**改动内容**：
1. 新增辅助函数 `_transfer_amount()` [settlement.py#L48-L54]：
   ```python
   def _transfer_amount(*, base: int, payer: int, receiver: int, dealer: int, rules: RulesConfig) -> int:
       if not rules.enable_dealer_multiplier:
           return base
       if payer == dealer or receiver == dealer:
           return base * int(rules.dealer_multiplier)
       return base
   ```

2. `settle_hu()` 新增 `dealer` 参数 [settlement.py#L81]:
   - 所有转账均通过 `_transfer_amount()` 处理
   - 判断逻辑：若payer或receiver任一为庄家，则乘以倍数

3. `settle_gang()` 新增 `dealer` 参数 [settlement.py#L55-L76]:
   - 杠分同样应用庄闲倍数逻辑

   enable_dealer_multiplier: bool = True

## 三、代码质量分析

### 架构设计评价

#### ✅ 优点

1. **关注点分离清晰**：
   - `_transfer_amount()` 独立处理庄闲倍数逻辑
   - settle_* 函数职责单一（仅处理结算分配）
   - 规则由 RulesConfig 集中管理

2. **参数变更向后兼容**：
   - 虽然签名变更（source -> self_draw/from_player），但调用处全部更新
   - 新增参数均有明确含义（dealer, self_draw）

3. **规则灵活性高**：
   - 支持 enable_dealer_multiplier 开关
   - 支持 dianpao_pays_all_alive 两种结算模式
   - 易扩展其他规则变种

4. **类型安全**：
   - 使用 `list[int]` 和 `int | None` 明确参数类型
   - 函数签名清晰（使用keyword-only参数）

#### ⚠️ 需注意的设计决策

1. **payer/receiver 双判断**（[settlement.py#L53](src/mahjong_ai/scoring/settlement.py#L53)）：
   ```python
   if payer == dealer or receiver == dealer:
       return base * int(rules.dealer_multiplier)
   ```
   - **设计**：任一方为庄家就×倍数
   - **对应川麻规则**：正确（庄家参与就×2）
   - **可能性**：无多余场景（不存在payer和receiver同时为dealer的情况）

2. **dianpao_pays_all_alive 行为**（[settlement.py#L103-L108](src/mahjong_ai/scoring/settlement.py#L103-L108)）：
   - 当True时，模拟"相对自摸"的赔付模式（点炮者赔付如三人都赔的总和）
   - 当False时，仅点炮者赔一个胡者的金额
   - **审核意见**：两种模式都正确，但False是川麻标准规则

3. **alive_before 使用一致**（[engine.py#L544](src/mahjong_ai/core/engine.py#L544)）：
   - RESPONSE 和 RESPONSE_QIANGGANG 都使用胡牌前的alive列表
   - **正确性**：✅ 确保零和约束不破坏
