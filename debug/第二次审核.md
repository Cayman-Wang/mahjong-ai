## 二、川麻一炮多响与庄闲倍数规则（已确认）

根据讨论确认的川麻规则：

### 一炮多响规则
1. **赔付主体唯一**：仅点炮者赔，其他未胡玩家不支付
2. **分别独立结算**：点炮者按每个胡者的番数分别计算，**互不影响**（不乘以胡者人数）
3. **番数正常计算**：胡牌玩家的番数不受多响影响
4. **不立刻结束**：一炮多响后游戏继续进行，直到3人胡或流局

### 庄闲倍数规则
- **庄家参与任何赔付均×2**：
  - 庄家自摸：闲家各赔 `底分×2^番数×2`
  - 庄家点炮给闲家：庄家赔 `底分×2^番数×2`
  - 闲家点炮给庄家：闲家赔 `底分×2^番数×2`
  - 闲家自摸：庄家赔 `底分×2^番数×2`，其他闲家赔 `底分×2^番数`

### 结算示例
**场景**：玩家0（庄）点炮，玩家1（3番）和玩家2（4番）同时胡牌
- 玩家1得分：`底分×2^3×2 = 底分×16`（庄家参与×2）
- 玩家2得分：`底分×2^4×2 = 底分×32`（庄家参与×2）
- 玩家0扣分：`-(底分×16 + 底分×32) = -底分×48`
- 玩家3得分：0（未参与）
- 零和验证：`16 + 32 - 48 + 0 = 0` ✓

## 三、修改方案详情

### 方案1：修改 `settle_hu()` 函数签名

**当前签名**（[settlement.py](src/mahjong_ai/scoring/settlement.py#L63)）：
```python
def settle_hu(
    winner: int,
    source: int | None,
    fan_total: int,
    alive: list[bool],
    rules: RulesConfig
) -> list[int]:
```

**目标签名**：
```python
def settle_hu(
    winner: int,
    source: int | None,
    fan_total: int,
    alive: list[bool],
    rules: RulesConfig,
    dealer: int,                    # 新增：庄家位置
    is_multiple_hu: bool = False    # 新增：是否一炮多响
) -> list[int]:
```

**修改逻辑**：
1. 计算基础点数：`pts = 底分 × 2^fan_total`
2. 判断庄闲倍数：
   - 若 `winner == dealer` 或 `source == dealer`，则 `pts *= 2`
3. 赔付逻辑：
   - **自摸**（`source is None`）：
     - 其他存活玩家各赔 `pts`（已有庄闲倍数）
   - **点炮**（`source is not None`）：
     - 若 `is_multiple_hu=False`：点炮者赔 `pts × 存活赔付者数量`（保持现有逻辑）
     - 若 `is_multiple_hu=True`：点炮者**仅赔 `pts`**（修正一炮多响问题）

## 四、设计决策点

### 4.1 alive_before 时机
- **确认**：RESPONSE 和 RESPONSE_QIANGGANG 使用 `alive_before`（胡牌前状态）
- **原因**：零和约束要求所有参与结算的玩家在胡牌前存活

### 4.2 零番胡牌处理
- **确认**：保持现有 `fan_total = max(fan_total, 1)` 逻辑
- **原因**：底分×1 确保最低得分，防止零分胡牌

### 4.3 分数累加顺序
- **确认**：保持 `state.scores[i] += score_delta[i]` 累加模式
- **原因**：支持多次结算（一炮多响）的分数叠加

### 4.4 庄家轮转
- **当前状态**：`dealer` 字段存在但无轮转逻辑
- **决策**：本次修改仅使用现有 `dealer` 计算倍数，不实现轮转
- **未来扩展**：可在 round_end 时实现"点炮者做庄"或"连庄"规则

@staticmethod
def _pick_closest_to_player(candidates: list[int], reference: int) -> int:
    """从候选玩家中选择顺位最接近参考玩家的一个"""
    if not candidates:
        raise ValueError("候选列表不能为空")
    candidates_sorted = sorted(candidates, key=lambda p: (p - reference) % 4)
    return candidates_sorted[0]
